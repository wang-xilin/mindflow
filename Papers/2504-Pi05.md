## Summary

Physical Intelligence 提出 [[π0.5]]：在 [[π0]] 基础上，通过把 mobile manipulation、跨 embodiment、web 数据与 high-level 语义标注**统一 co-training**，并在推理时显式做"高层子任务预测 → 低层 action 生成"的层级分解，首次让端到端 [[VLA]] 在**完全未见过的真实家庭**中完成 10–15 分钟的长时域灵巧操作。

**Key Takeaways:**
1. **Co-training is the moat**：97.6% 的训练数据并非来自目标 mobile manipulator，但去掉任何一个异构来源（ME、CE、WD、HL）都显著掉点 —— data diversity ≫ data specificity。
2. **Hybrid action representation**：pre-training 用 [[FAST]] discrete tokens（高效且与 VLM 同源），post-training 才接 [[Flow Matching]] action expert 拿到精细控制，绕开了 continuous head 在大规模异构数据上不稳的痛点。
3. **Hierarchical inference**：同一个模型先 autoregressive 生成子任务字符串（如 "pick up the plate"），再 condition 在子任务上去 sample action chunk —— 比直接 flat policy 或外接 GPT-4 规划都强。
4. **Environment scaling 收敛**：训练环境从 3 → 104 时性能逼近"训练集就包含测试家庭"的 oracle，说明真实家庭数据的边际收益还没饱和但已能跨越泛化鸿沟。

## Problem & Motivation

让机器人真正"进入千家万户"需要 **open-world generalization**：在从未见过的房间、面对从未见过的物体、依然能完成长时域操作。但现有 robot policy 几乎都被锁在训练过的环境里，原因有三：

1. 单一来源的 robot data 远不足以覆盖真实家庭的视觉/物理多样性；
2. 长时域任务（清洁厨房、整理卧室）需要语义规划与精细控制的协同，但二者通常被切成两个系统；
3. 不同 embodiment 之间的知识无法共享，每加一个 robot 就要重新攒数据。

π0.5 的赌注是：**用一个统一模型吃下所有能拿到的异构数据**，让 VLM 的 web 知识、其他 robot 的操作经验、人工语义标注都来补 mobile manipulator 数据的短板。

## Method

π0.5 是一个 multimodal transformer，输入为 image patches、language tokens 与 robot state，输出既有 text token（子任务名、bbox、VQA 回答），也有 continuous action chunk。整体由三块拼成：层级化推断、混合 action 表示、异构 co-training。

**Figure 3. 模型总览：pre-training 用 discrete token 同时训练所有数据源；post-training 接入 flow matching action expert 做精细控制**

![architecture](https://arxiv.org/html/2504.16054v1/x2.png)

### Hierarchical Inference

模型把 policy 写成"先想再做"两步：

**Equation 1. Joint policy 分解**

$$
\pi_\theta(\mathbf{a}_{t:t+H}, \hat{\ell} \mid \mathbf{o}_t, \ell) \;=\; \pi_\theta(\mathbf{a}_{t:t+H} \mid \mathbf{o}_t, \hat{\ell}) \cdot \pi_\theta(\hat{\ell} \mid \mathbf{o}_t, \ell)
$$

**符号说明**：$\mathbf{o}_t$ 当前观测（多视角 RGB + state），$\ell$ 用户给的高层指令（如 "clean the kitchen"），$\hat{\ell}$ 模型自回归生成的当前子任务（如 "pick up the plate"），$\mathbf{a}_{t:t+H}$ 长度为 $H$ 的 action chunk。

**含义**：同一个 transformer 既扮演 high-level planner（条件 $\ell$ 输出 $\hat{\ell}$），又扮演 low-level controller（条件 $\hat{\ell}$ 输出 action）。两个角色共享 backbone，使得 web/语义数据训练得到的"语义理解"能直接喂给控制头。

### Hybrid Discrete-Continuous Action

Action 同时被两套 head 监督：autoregressive [[FAST]] token head（pre-training 主力）+ flow matching action expert（post-training 切入）。Flow matching 把 noisy action 沿直线插值回 ground truth：

**Equation 2. Flow matching 路径**

$$
\mathbf{a}_{t:t+H}^{\tau,\omega} \;=\; \tau\,\mathbf{a}_{t:t+H} + (1-\tau)\,\omega, \quad \omega \sim \mathcal{N}(0,\mathbf{I}),\;\tau \in [0,1]
$$

**符号说明**：$\omega$ 高斯噪声，$\tau$ 流时间索引，沿 $\tau$ 从 0 到 1 把噪声"流"回真 action。

**Equation 3. 联合 pre/post-training 目标**

$$
\mathbb{E}_{\mathcal{D},\tau,\omega}\!\left[\, H\!\big(x_{1:M},\, f^{\ell}_\theta(\mathbf{o}_t,\ell)\big) \;+\; \alpha \,\big\|\,\omega - \mathbf{a}_{t:t+H} - f^{a}_\theta(\mathbf{a}^{\tau,\omega}_{t:t+H},\mathbf{o}_t,\ell)\,\big\|^2 \,\right]
$$

**符号说明**：$H(\cdot,\cdot)$ token 交叉熵（既覆盖语言/子任务输出 $x_{1:M}$ 也覆盖 FAST action token），$f^{a}_\theta$ flow matching action expert，$\alpha$ 两个 loss 的权重。

**含义**：Pre-training 阶段 $\alpha=0$，纯 token 损失，280k steps —— 这一步让所有异构数据（图文、跨 embodiment、web VQA）都能用统一的 next-token 目标喂进来。Post-training 80k steps 才把 $\alpha$ 设为 10.0 启用 flow matching expert，让 mobile manipulator 数据上的精细动作"接管"输出。这种切换避开了 continuous regression head 在多源异构数据上不稳的老问题，同时拿到了 flow matching 的高频精细控制（50 Hz，10 步 denoising）。

### Heterogeneous Co-training Recipe

| 类别 | 内容 | 占比/规模 |
|---|---|---|
| **MM** Mobile Manipulator | 目标平台在 ~100 个真实家庭采集的家务 demonstration | ~400 小时 |
| **ME** Multi-Environment | 固定臂在多样家庭/办公环境的操作数据 | 大规模 |
| **CE** Cross-Embodiment | 实验室多种机器人 + [[Open X-Embodiment]] | 大规模 |
| **HL** High-Level | 人工标注的子任务序列（用作 $\hat{\ell}$ 的监督） | 多任务覆盖 |
| **WD** Web Data | image captioning, VQA, object detection/grounding | 互联网级 |
| **VI** Verbal Instructions | 专家在采集时口头指挥 robot 的语言 trace | 训练样本的 ~11% |

关键数字：**只有 2.4% 的训练样本来自目标设定（mobile manipulator 做家务）**。其余全是"间接"数据，但消融显示它们才是泛化的主因。

### 硬件平台

双臂 mobile manipulator：每臂 6-DOF + 平行夹爪，4 个 RGB 相机（双腕 + 前后），全向轮底盘 + torso lift；50 Hz 端到端控制，无显式 planner 或碰撞检测。

## Experiments

### Datasets & Setup

| 数据 | 规模 | 用途 |
|---|---|---|
| MM 家务数据 | ~400 小时, ~100 homes | 主任务 demonstration |
| ME / CE | 多环境 + OXE | Co-training 异构源 |
| Web 图文 | CapsFusion / COCO / VQAv2 等 | 语义先验 |
| **Mock homes** | 受控复现的厨房/卧室 | 量化对比与消融 |
| **3 个真实新家** | 训练中从未出现 | 长时域任务最终评测 |

### Implementation Details

- Backbone：基于 [[π0]] 的 PaliGemma 风格 multimodal transformer + flow matching action expert
- Pre-training：280k steps, $\alpha=0$（纯 token loss）
- Post-training：80k steps, $\alpha=10.0$（接入 flow matching）
- Action chunk 长度 $H$，10 步 denoising，50 Hz 控制
- 评测任务：dishes-in-sink、items-in-drawer、laundry、make-the-bed 等长时域多阶段家务

## 论文点评

### Strengths

1. 首次端到端实现"完全没见过的真实家庭 → 10–15 分钟长时域家务"，把 VLA 从 demo 推到了真实泛化的门槛上。
2. Hybrid token/flow 训练把"VLM 风格大规模 pretrain"与"flow matching 精细控制"自然拼起来，工程上是个值得复用的 recipe。
3. 消融非常彻底，把每一类数据的贡献剥得清清楚楚 —— ME/CE/HL/WD/VI 各自定位明确，几乎没有"摆设"数据。
4. 同一模型既做 high-level 规划又做 low-level 控制，避免了 LLM-planner + 低层 policy 的接口失配，并且实验直接打脸"外挂 GPT-4 当 planner"。
5. 真机部署、3 个真实新家 ×4 任务 ×多 trial，证据强度远超 sim-only 工作。

### Weaknesses

1. 结果几乎全是 bar chart，**没有正式的数字 table**，对复现和细粒度对比不友好。
2. 真实家庭只测了 3 个，且都是"住宅风格"家庭，没有涉及更极端的视觉/光照/物体分布；声称的 open-world 仍是有限分布。
3. 失败模式诚实但代价高：陌生硬件（怪把手、卡住的柜门）、手臂遮挡、长时序 high-level 抖动 —— 都是 deployment 的硬伤。
4. 没有 cross-room navigation、空间记忆，长时域规划只在单房间内成立；context window 也很短。
5. 对外部研究者基本不可复现：核心壁垒是 100+ 真实家庭的数据采集和 mobile manipulator 平台，论文也没公开权重/数据/训练代码。
6. "97.6% 不是目标数据"的说法很性感但容易被滥读 —— 不能脱离那 2.4% 的高质量 in-domain 数据来谈泛化。

## Notes

- 97.6% 训练数据不是目标任务 —— 这是支持 **"data diversity ≫ data specificity"** 的最强 case study 之一，但要小心：那 2.4% in-domain 数据是必要而非充分条件。
- Hybrid action representation 的真正价值是**让异构 token 数据和精细控制共用一个 backbone**，而不是单纯"两阶段训练"。这套思路可能比 hierarchical inference 更具复用性。
- 让 VLA 自己生成 subtask 比外挂 GPT-4 强，原因不是 GPT-4 笨，而是它无法实时感知 affordance（柜门是否开着、手是否空着）。这对所有"LLM-as-planner"路线都是个警告：规划必须 grounded 在当前 embodiment state 上。
- HL 监督带来 25% → 65% 的巨大跃升，意味着"语义子任务标注"的 ROI 极高 —— 比再标 robot trajectory 划算得多。这是个值得借鉴的数据策略。
- Physical Intelligence 的真正护城河是 100+ 真实家庭的数据采集 pipeline；学术界要复刻基本无望，但 recipe 本身（hybrid token/flow + heterogeneous co-training + hierarchical inference）是可以借鉴到更窄的 setting 的。
- 与 [[UWM]]、[[EmbodiedR]] 等同期 VLA/world model 工作放一起看：π0.5 押的是 **"足够多的异构真数据 + 简单架构"**，而不是 world model 或 RL —— 这是一个值得追踪的路线分歧。
