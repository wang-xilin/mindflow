## Summary
π0.5 在 [[2410-Pi0|π0]] 基础上引入异构数据 co-training（跨 embodiment 机器人数据 + 高层语义子任务预测 + 网络多模态数据），首次端到端地在训练中未见过的真实家庭中完成长时程、灵巧的移动操作任务（如清理整个厨房和卧室）。

**Key Takeaways:**
1. **Heterogeneous co-training is the main lever**: 单靠机器人数据不足以 open-world 泛化，跨 embodiment 机器人 (ME+CE)、网络数据 (WD)、高层语义子任务 (HL) 全部参与 co-training 才能在真实家庭泛化，移除任一数据源都显著掉点。
2. **Hybrid discrete + continuous action head**: 同一个模型既用 FAST 离散 action token（pre-training 友好，可与 language/subtask token 同 loss 训练），又接 flow matching 连续 action expert（inference 时高频灵巧控制），两阶段 curriculum 让二者共存。
3. **High-level subtask prediction 是桥梁**: 模型在 inference 时先预测自然语言子任务 `ˆℓ` 再条件化输出低层动作，这一显式中间层既让训练能吸收人类语言标注数据，也让 long-horizon 任务可 decompose。
4. **泛化 scaling law**: generalization 随训练场景数量平滑提升；从 ~100 个环境起，模型已能在全新家庭达到接近 "训练分布内" baseline 的成功率。
5. **实证清单**：整厨房清理、卧室清理等长时程任务在全新、未见过的 Airbnb 民宅中完整执行，是当前 end-to-end VLA 在真实家庭泛化上报告到的最强结果。

## Introduction

Open-world generalization 是 embodied AI 最核心的未解问题：机器人只有离开实验室、能够处理真实世界的 long-tail 才算真正有用。π0.5 的立场是——单纯 scale 同源机器人数据不够，必须引入**多层次异构数据 co-training**：直接相关的同平台操作数据、不同 embodiment 的迁移数据、以及完全不同模态的语言和视觉任务数据。这样模型才能在语义（该打开哪个抽屉）、物理（没见过的形状如何抓）、时序（长任务如何分解）三个层次同时泛化。

**Figure 2. π0.5 cleaning a new kitchen.** The robot is tasked with cleaning a kitchen in a home that was not in the training data. The model is given general tasks (close the cabinets, put the items in the drawer, wipe the spill, and put the dishes in the sink), which it performs by both predicting subtasks to accomplish (e.g., pick up the plate) and emitting low-level actions.
![](https://arxiv.org/html/2504.16054v1/x1.png)

## Preliminaries

论文把 VLA 建模成一个统一的条件序列模型：给定观测 $\mathbf{o}_t$ 和任务指令 $\ell$，目标是输出动作块 $\mathbf{a}_{t:t+H}$。作者把 [[2410-Pi0|π0]] 的 flow matching action head 保留下来作为低层控制器，但在 pre-training 阶段把低层动作统一 tokenize（借 FAST 离散化），从而让动作和语言 / subtask / 图像这些异构 token 共享同一个 transformer 的训练目标。这为下一章的 hybrid head 铺路。

## IV The π0.5 Model and Training Recipe

**Figure 3. Model overview.** π0.5 is trained in two stages: (i) a pre-training stage that combines heterogeneous sources (multi-robot data, high-level semantic subtask prediction, web data) with actions represented as FAST discrete tokens; (ii) a post-training stage that specializes the model for downstream deployment with a flow matching action expert.
![](https://arxiv.org/html/2504.16054v1/x2.png)

### IV-A Architecture

模型在推理时把联合分布分解为一个**高层语言子任务预测**和一个**低层动作预测**：

$$
\pi_\theta(\mathbf{a}_{t:t+H}, \hat{\ell} \mid \mathbf{o}_t, \ell) = \pi_\theta(\hat{\ell} \mid \mathbf{o}_t, \ell)\,\pi_\theta(\mathbf{a}_{t:t+H} \mid \mathbf{o}_t, \hat{\ell})
$$

**符号说明**：$\mathbf{o}_t$ 当前观测，$\ell$ 人类给的高层指令，$\hat{\ell}$ 模型自己预测的自然语言子任务（如 "pick up the plate"），$\mathbf{a}_{t:t+H}$ 长度为 $H$ 的动作 chunk。
**含义**：模型先把大任务切成可执行的语言子任务，再把子任务作为短时条件化信号生成低层动作，本质上是一个显式的 hierarchical policy。

### IV-B Combining discrete & continuous action representations

Pre-training 和 inference 对动作表示的要求是冲突的：pre-training 想把动作和其他 token 一起做 autoregressive next-token loss（因此要离散），但 inference 想要高频连续控制（因此要 flow matching）。π0.5 的解法是**同一个 VLA backbone 接两个 action head**：一个 FAST 离散 head 吃 cross-entropy loss，一个 flow matching head 吃 flow matching loss，二者 loss 相加：

$$
\mathcal{L}_{\pi_{0.5}} = \mathbb{E}_{\mathcal{D},\tau,\omega}\Big[H\big(x_{1:M}, f^\ell_\theta(\mathbf{o}_t, \ell)\big) + \alpha \big\lVert \omega - \mathbf{a}_{t:t+H} - f^a_\theta(\mathbf{a}^{\tau,\omega}_{t:t+H}, \mathbf{o}_t, \ell)\big\rVert^2 \Big]
$$

**符号说明**：$H(\cdot,\cdot)$ cross-entropy，$f^\ell_\theta$ 输出离散子任务 / FAST token logits，$f^a_\theta$ 是 flow matching velocity 网络，$\mathbf{a}^{\tau,\omega}$ 是加噪的动作 chunk，$\alpha$ 是两个 loss 之间的权重。
**含义**：同一个 transformer 既学语言 / 子任务 / 图像这种离散序列预测，也学连续动作去噪场。Pre-training 侧重离散侧（$\alpha$ 小或为 0），post-training 的 $\alpha$ 拉起来让 flow matching head 收敛。

### IV-C Pre-training

Pre-training 混合了多种数据源：

- **MM (Mobile Manipulators)**：Physical Intelligence 自家移动双臂机器人上采集的同平台数据。
- **ME (Multi-Environment non-mobile)**：非移动机器人在多样环境里采集的数据，带来环境多样性。
- **CE (Cross-Embodiment)**：来自其他 embodiment 的实验室数据，带来形态多样性。
- **HL (High-Level)**：人工标注的任务分解——给一段视频 + 高层指令，标出该时刻应执行的子任务 $\hat{\ell}$。
- **WD (Web Data)**：多模态网络数据（图像 + 语言任务），带来 open-world 语义先验。

所有动作都过 FAST action tokenizer 离散化，使动作和语言 token 共享 autoregressive loss。

**Figure 4. Examples from pre-training and post-training tasks.** π0.5 is pre-trained on data from mobile manipulators (MM), non-mobile robots in diverse environments (ME), and cross-embodiment data (CE), plus high-level subtask prediction (HL) and multi-modal web data (WD). Post-training additionally uses verbal instructions (VI) and drops the lab CE data.
![](https://arxiv.org/html/2504.16054v1/x3.png)

### IV-D Post-training

Post-training 阶段切换到部署相关的数据切片：保留 MM + HL + WD，新增 **VI (Verbal Instructions)**——来自人类口头指导的子任务序列，去掉 CE（CE 任务与部署目标 embodiment 不同，收益递减）。同时把 $\alpha$ 拉起来让 flow matching head 以高频高精度方式接管低层控制。

### IV-E Robot system details

**Figure 5. Robot system overview.** Two mobile manipulator platforms — each has four cameras (forward, backward, and both wrists), two 6-DoF arms with parallel jaw grippers, a mobile base, and a torso lift mechanism. The π0.5 model controls the joints and grippers of each arm, base velocity, and lift, resulting in 18–19 DoF state and action spaces.
![](https://arxiv.org/html/2504.16054v1/x4.png)

### V-B How does generalization scale with the number of scenes?

核心 scaling 问题：训练场景数量怎样影响对全新场景的泛化？作者逐步增加训练环境数量，测量模型在 held-out mock homes 上的 task success rate 和 language-following rate。

### V-C How important is each part of our co-training recipe?

逐个移除 pre-training 数据源做 ablation。结论是 cross-embodiment 类数据（ME + CE）对总体成功率至关重要；web data (WD) 的收益主要体现在 out-of-distribution 语言跟随上。

**Figure 10. Training recipe ablations on mock homes.** Variants that exclude different parts of the training mixture on all four test tasks (10 trials per policy and task). Removing either ME or CE (or both) causes large degradation. Web data has limited effect on in-distribution success but is important for OOD.
![](https://arxiv.org/html/2504.16054v1/extracted/6379337/figures/performance_vs_ll_data.png)

**Figure 11. Training recipe ablations, language following.** In-distribution vs. OOD language following with/without each data source. Web data (WD) is particularly important for OOD language following; CE and ME both affect both splits.
![](https://arxiv.org/html/2504.16054v1/extracted/6379337/figures/data_vs_generalization.png)

### V-D How does π0.5 compare to other VLAs?

在相同 mock-home 设置下对比 π0.5、[[2410-Pi0|π0]]、π0-FAST+Flow 三个模型。π0.5 全面胜出，说明性能提升主要来自异构 co-training 而不是单纯的 architecture 调整。

**Figure 12. Comparing π0.5 with other models.** The full π0.5 significantly outperforms both [[2410-Pi0|π0]] and π0-FAST+Flow in the mock home test environments.
![](https://arxiv.org/html/2504.16054v1/extracted/6379337/figures/performance_vs_ll_model.png)

### V-E How important is high-level inference?

最后一个问题：推理时显式先预测 $\hat{\ell}$ 再条件化低层动作（"explicit HL"）是否必要？作者对比三种设定：完整 π0.5（explicit HL）、只跑低层（implicit HL）、以及在训练中去掉 HL 数据。

**Figure 13. Evaluation of the high-level inference process.** The full π0.5 with explicit high-level + low-level inference attains the best results. Using only low-level inference ("implicit HL") on the full π0.5 still benefits from HL examples in training. Excluding HL and verbal-instruction data hurts performance substantially.
![](https://arxiv.org/html/2504.16054v1/extracted/6379337/figures/performance_vs_hl_data.png)

## 论文点评

### Strengths

1. **问题定义直击痛点**：open-world 家庭部署是 VLA 方向最能把 paper 和 product 拉齐的问题设定，而且 "在 Airbnb 民宅里自主清理卧室" 这种 demo 的信息量远高于实验室 benchmark。
2. **Co-training 的 ablation 非常系统**：不是模糊地说 "异构数据有用"，而是给出 MM / ME / CE / HL / WD / VI 六类数据各自的贡献 breakdown，移除任一类的代价都可以读出来，方便别人复用该训练食谱。
3. **Hybrid discrete + continuous head 是干净的工程方案**：把 "pre-training 想要离散 token、inference 想要高频连续" 这个老矛盾用两阶段 $\alpha$ 调度 + 共享 backbone 解决，而不是训两个模型或者两阶段蒸馏。
4. **开源覆盖完整**：`Physical-Intelligence/openpi` 同时支持 inference 和 training，base checkpoint (`pi05_base`) 与下游 (`pi05_libero`, `pi05_droid`) fine-tuned 权重都对外发布，这在 frontier VLA 里非常少见。

### Weaknesses

1. **成功率的绝对数依然不公开透明**：正文全部以 bar chart 呈现结果，主表级的数字没有汇总成 table，第三方再现性评估需要先肉眼读图。
2. **"新家庭" 的分布偏移量未量化**：作者强调测试在未见 Airbnb 民宅，但没有刻画训练和测试环境分布的差异（比如光照、相机视角、家具 class 分布），所以 "泛化到新家庭" 的难度无法和其他工作横向比较。
3. **Web data 的角色可能被 overstated**：Figure 11 显示 WD 对 OOD 语言跟随重要，但对 in-distribution 成功率影响有限；正文措辞偏向 "WD 很重要"，实际是 "WD 在语言 grounding 上有增量"。
4. **VI (口头指令) 的数据规模和来源未披露细节**：post-training 新增的 verbal instruction 数据缺少组成、规模和采集协议描述。
5. **Long-horizon 失败模式的分析不足**：视频能看到一些失败 case（比如放错位置、漏掉物品），但 paper 没有做失败模式分类或 root-cause 分析。

### 可信评估

#### Artifact 可获取性
- **代码**: inference + training 都开源（`Physical-Intelligence/openpi`，JAX 为主，2025 年 9 月补齐 PyTorch 推理/微调）。
- **模型权重**: 已发布 `pi05_base`（基础 fine-tuning 起点）、`pi05_libero`（LIBERO benchmark，SOTA）、`pi05_droid`（DROID 数据微调，快速推理 + 良好语言跟随）；同 repo 还附带 `pi0_base` / `pi0_fast_base` 及若干 ALOHA / DROID fine-tuned checkpoint。
- **训练细节**: 正文给出 co-training 数据类别（MM/ME/CE/HL/WD/VI）和两阶段 $\alpha$ 调度概念，但完整的超参、数据比例、训练步数没有系统披露；openpi repo 的 training configs 里能读到部分 hyperparameter。
- **数据集**: 主要为 Physical Intelligence 内部采集的 mobile manipulator 数据（私有）；public 组件包括 DROID、LIBERO、以及其它 cross-embodiment 公开数据集。完整的 co-training mixture 没有对外发布。

#### Claim 可验证性
- ✅ **"能在未训练过的真实家庭完成长时程任务"**：project page 有多段完整 rollout 视频，包括 Airbnb 民宅的 fully-autonomous 卧室清理（见 V-A 嵌入的 video）；weights 开源，理论上可复现。
- ✅ **"移除异构数据会显著掉点"**：V-C 的 ablation 给了逐项对比，实验设置清楚。
- ⚠️ **"泛化随场景数平滑 scale"**：Figure 8/9 的 x 轴点数不算密，低数据区间的不确定性可能被平滑掉；作为 scaling 趋势参考可以，作为 scaling law 还不够严格。
- ⚠️ **"首次端到端 VLA 能在新家庭完成长时程灵巧操作"**：这个 "first" claim 受限于同期工作的定义边界（例如某些带 high-level planner 的系统是否算 end-to-end）；需要把 "end-to-end" 的范围讲清楚才能判断。
- ⚠️ **"Web data 对 OOD 泛化很关键"**：只在语言跟随 OOD 这一个切面上成立，正文表述比 ablation 能支持的结论更强。

## 关联工作

### 基于
- **[[2410-Pi0|π0]]**：π0.5 的直接前身，共享 PaliGemma + flow matching action expert 的主体结构，π0.5 的增量是 co-training 数据混合和 hybrid discrete/continuous head。见 Papers/2410-Pi0.md。
- **FAST action tokenizer**：把连续动作离散化为 token，使 pre-training 能和语言 / subtask 共享 autoregressive loss。π0.5 用它承担 pre-training 阶段的动作表示。

### 对比
- **[[2410-Pi0|π0]]**：同实验室前作，作为 V-D 的主要 baseline；π0.5 在 mock home 上全面领先，说明异构 co-training 的增量。
- **π0-FAST**：[[2410-Pi0|π0]] 的 autoregressive 变体，作为额外的 FAST-only 对照。
- **[[2406-OpenVLA|OpenVLA]]**：mock home 实验中作为外部 VLA baseline（论文 Related Work 对标的主流 VLA 之一）。见 Papers/2406-OpenVLA.md。
- **[[2405-Octo|Octo]]**：早期通用机器人 policy，作为 cross-embodiment baseline 之一。见 Papers/2405-Octo.md。

### 方法相关
- **Flow Matching**：低层动作表示的生成目标，[[2410-Pi0|π0]] 首次引入到 VLA，π0.5 保留并与离散 action token 并用。
- **Hierarchical high-level planning**：通过显式预测自然语言子任务分解长时程任务，概念上接近于"VLM planner + low-level policy"但训在同一个网络里。
- **Cross-embodiment training (RT-X / OXE 风格)**：把跨平台机器人数据放进单一 policy 来换泛化。π0.5 的 CE 部分本质是同一思路。
